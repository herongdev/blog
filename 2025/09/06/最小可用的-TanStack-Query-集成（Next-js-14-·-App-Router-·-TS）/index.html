5) 与你现有封装的关系

  • 无需修改 http.call / client.ts / cancel.ts；Vue Query 只是把“请求何时发起、如何缓存/重试/并发合并”的状态机托管出去。
  • 其他简单页面仍可直接用 http.get/post 或 http.call。把 “读多写少、复用度高”的接口（比如价格趋势、列表）逐步迁到 Vue Query 即可，按需引入、零破坏。
