## 三、网络安全与信息安全管理制度

### 1. 网络安全防御措施

#### 1.1 部署 WAF

- **1.1.1** 部署了 WAF（Web 应用防火墙）或使用托管平台提供的安全防护
- **1.1.2** 防止 SQL 注入、XSS 攻击等常见网络攻击

#### 1.2 更新补丁

- **1.2.1** 定期更新操作系统和应用程序补丁（如使用云服务器）
- **1.2.2** 定期更新 VitePress 及相关依赖包
- **1.2.3** 防止已知漏洞的利用

#### 1.3 流量监控

- **1.3.1** 对网站流量进行监控
- **1.3.2** 识别并拦截异常访问请求
- **1.3.3** 使用 CDN 服务提升访问速度和安全性

#### 1.4 HTTPS 加密

- **1.4.1** 全站启用 HTTPS 加密传输
- **1.4.2** 使用有效的 SSL/TLS 证书

### 2. 信息安全管控制度

#### 2.1 数据加密

- **2.1.1** 网站为静态站点，不涉及用户敏感数据存储
- **2.1.2** 如涉及用户数据，采用加密存储
- **2.1.3** 敏感信息（如密码）采用哈希算法处理

#### 2.2 权限控制

- **2.2.1** Git 仓库访问权限严格控制
- **2.2.2** 只有授权人员可以访问和修改网站内容
- **2.2.3** 服务器访问权限严格控制

#### 2.3 数据备份

- **2.3.1** 定期备份网站源代码和内容文件
- **2.3.2** 备份数据存储在安全的云存储或 Git 仓库中
- **2.3.3** 建立版本控制机制，便于内容恢复

#### 2.4 内容审核

- **2.4.1** 发布前对技术文章内容进行审核
- **2.4.2** 确保内容符合法律法规要求
- **2.4.3** 不发布违法违规内容

### 3. 应急处理方案

#### 3.1 漏洞处理

- **3.1.1** 一旦发现安全漏洞或遭受攻击
- **3.1.2** 立即启动应急预案，封锁相关访问，查找并修补漏洞
- **3.1.3** 及时更新依赖包和系统补丁

#### 3.2 服务器切换

- **3.2.1** 如果网站出现不可用情况
- **3.2.2** 立即启动备用服务器或切换到备用托管服务进行切换，确保服务的持续性

#### 3.3 内容恢复

- **3.3.1** 如内容被恶意篡改，立即从备份恢复
- **3.3.2** 利用 Git 版本控制快速回滚到正常版本
