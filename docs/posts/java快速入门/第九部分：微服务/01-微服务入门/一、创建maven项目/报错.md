java: 程序包org.springframework.boot不存在的解决方法
最近IDEA抽风了，不管是新建SpringBoot工程，还是导入项目。
IDEA代码里面都会飘红~
Build项目时，会提示错误：**错误:(3, 32) java: 程序包org.springframework.boot不存在**
后来找到一个靠谱的解决方案，建议使用方法二。
方法一虽然也能解决问题，解决后项目能够正常启动，但是IDEA里导入包、变量引用这些地方还是会标红。
主要的原因就是：idea配置的[maven](https://so.csdn.net/so/search?q=maven&spm=1001.2101.3001.7020)加载不到autoconfigure。

![Exported image](Exported%20image%2020260703000915-0.png)

看着挺让人难受的，以下是两种解决方法：一个是修改idea配置，一个是手动加载maven的包
方法一
打开idea的设置，在mav的运行程序项里面，勾上【将IDE构建/运行操作委托给Maven】，如图：

![Exported image](Exported%20image%2020260703000920-1.png)

方法二
通过 mvn -U idea:idea 命令重新加载maven包，具体操作是这样的：打开cmd窗口cd 到 工程根目录，比如我的工程是：D:\IdeaProjects\demo，然后执行 mvn -U idea:idea 命令，完了以后重新运行项目就正常了！

![Exported image](Exported%20image%2020260703000923-2.png)
![Exported image](Exported%20image%2020260703000929-3.png)
