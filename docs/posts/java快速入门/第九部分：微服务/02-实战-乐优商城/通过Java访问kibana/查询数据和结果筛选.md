`普通查询：match用来做全文查询`

![GET heima4 search query match FIELD TEXT](Exported%20image%2020260702235835-0.png)

`结果，查询的时候不过对存储的值进行了分词存储，也对查询的关键词进行分词，然后再查询 。`

![GET heima4 query match title VIVO , type goods , t...](Exported%20image%2020260702235844-1.png)

如果要对结果进行限制，可以加上operator关键字，如设置为and，那分词后的关键词同时满足的显示出来。
![GET heima4 query match title search 1 3 4 6 7 8 11...](Exported%20image%2020260702235847-2.png)

- **多字段查询**
- multi_match与match类似，不同的是它可以在多个字段中查询

![GET heima4 qu multi title , subTitle type goods se...](Exported%20image%2020260702235849-3.png)

**词条查询**
term查询被用于精确匹配，这些精确值可能是数字、时间、布尔或者那些来分词的字符串，对搜索关键词不再进行分词后再查询，直接使用value设置的值进行搜索
![GET heima4 search query term title value xiaomil t...](Exported%20image%2020260702235851-4.png)

`对于不能分词的字段，效果是一样的`

![GET heima4 search query term type goods type goods...](Exported%20image%2020260702235854-5.png)
