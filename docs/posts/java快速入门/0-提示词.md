# Java 教程 0-提示词

把下面模板复制给 AI，可以把项目里的 Java 代码归类到 [Java 快速入门教程](./README.md) 的对应章节。目标不是只解释语法，而是补齐场景、边界、调用链、验证方式和常见错误。

---

## 基础分类

- `把这段 Java 代码的知识点分类到 java快速入门 教程中`
- `这段代码涉及哪些语法，放到对应的教程文件里`
- `解释这段 Java 代码并归类到合适的章节`
- `分析这个语法特性，添加到 Java 入门第 X 部分`

回答时必须说明：

- 它解决什么真实问题。
- 它属于哪一层：语法、模型、服务、数据访问、配置、测试或运维。
- 谁调用它，它又调用谁。
- 哪个命令、测试或接口请求可以证明它真的工作。

## 实战导向

- `以实战示例的形式，把这段 Java 代码加到教程`
- `用速成的方式解释这个 Java 语法，放到对应文件`
- `这是 Spring / Android 项目里的代码，分类到教程中`
- `把这个 Java 知识点做成速查表格式，写入 java快速入门`

## 推荐完整提示词

```text
请把下面这段 Java 代码归类到 docs/posts/java快速入门 系列：

1. 提取核心知识点，并说明它解决什么真实问题。
2. 判断它应该放入哪个章节和哪篇文章；如果需要新文章，请给出标题、front matter 和建议路径。
3. 说明它所在的层：语法、模型、服务、数据访问、HTTP 接口、配置、测试或运维。
4. 画出简短调用链，例如 Controller -> Service -> Mapper -> database。
5. 给出一个最小可运行示例或接口请求。
6. 给出验证方式：命令、预期输出、测试用例或错误复盘。
7. 标出常见坑，尤其是空值、类型转换、异常、SQL 参数、线程安全和版本差异。

代码：
（粘贴代码）
```

## 回答风格约定

- **自动识别归类**：多个知识点时分条归类
- **结构化**：适用场景 → 放入章节 → 调用链 → 核心代码 → 验证方式 → 常见坑
- **可运行**：示例尽量完整；普通 Java 示例保持 Java 17+ 可运行，新机器优先使用 LTS 版本
- **速成友好**：表格、列表优先，理论点到为止
- **不伪造结果**：没有运行过命令时，明确写“预期输出”，不要写成真实运行结果
