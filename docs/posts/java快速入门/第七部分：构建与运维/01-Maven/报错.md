再次执行maven package，BUILD SUCCESS

分析原因是因为：如果在plugin中添加maven-jar-plugin插件之后，maven会运行两次maven-jar-plugin插件，第一次运行是打包当前工程的jar，第二次运行是执行在plugin中设置的jar，这样的话会生成两个jar包，这里就要用到classifier这个属性了。classifier是用来标识不同的jar包的名称的一个分类符，这个属性的值会添加到工程jar包的名称后面。

一、现象：
运行mvn有关命令，报错“zsh: command not found: mvn”

二、原因
环境变量的配置文件没有生效。

三、解决方法
法一
使配置文件生效，使用如下命令。

source ~/.bash_profile
1
法二
现象：法一只能解决一次问题，因为重新打开终端，执行mvn相关命令，依然报错“zsh: command not found: mvn”，需要再执行source ~/.bash_profile才可以。
原因：zsh的配置文件没有加载~/.bash_profile的配置项。
解决方法：在zsh的配置文件中添加source ~/.bash_profile配置项的内容

vim ~/.zshrc
1
添加如下内容：

source ~/.bash_profile
1
