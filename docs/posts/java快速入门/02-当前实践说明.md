# 02-当前实践说明

这套笔记来自长期积累，里面既有仍然有用的基础知识，也有带明显年代感的框架和工具。学习时先按本页的实践边界取舍，再进入具体文章。

## 版本选择

| 主题 | 当前建议 | 学习时怎么用 |
|------|----------|--------------|
| Java | 新机器优先装 Java 21 或 Java 25 这样的 LTS 版本；练习示例保持 Java 17+ 可运行 | 基础语法不追逐新特性，先保证能编译、能测试、能解释 |
| Spring Boot | 新项目至少按 Java 17+ 思考；具体版本跟公司或项目一致 | 旧文章可帮助理解概念，实际新项目优先看官方文档 |
| Maven | 用 `pom.xml` 管理构建、依赖、测试和打包 | 不只记命令，要能解释依赖从哪里来、生命周期跑了什么 |
| JUnit | 新练习使用 Jupiter 风格测试；项目版本跟 Spring Boot 依赖管理保持一致 | 每个阶段至少有一个可重复运行的测试或验收命令 |

## 官方来源记录

| 来源 | 它说明了什么 | 本课程采用的取舍 |
|------|--------------|------------------|
| [Oracle Java SE Support Roadmap](https://www.oracle.com/java/technologies/java-se-support-roadmap.html) | Java 8、11、17、21、25 是 LTS 版本，后续 LTS 按路线继续发布 | 导读不再把 Java 8 当作唯一默认版本；新学习环境按 Java 17+，更推荐 LTS |
| [Oracle Java 25 Release Notes](https://www.oracle.com/java/technologies/javase/25-relnote-issues.html) | JDK 25 是 Java SE 25 平台的完整实现，并包含开发、调试、监控相关 API | 课程不强迫使用 Java 25 新特性；只把它作为当前 LTS 选择之一 |
| [Spring Boot System Requirements](https://docs.spring.io/spring-boot/system-requirements.html) | Spring Boot 当前文档要求 Java 17 或更高版本，并给出 Maven、Gradle 支持范围 | Spring Boot 章节按 Java 17+ 作为底线，旧 Spring 文章用于理解，不作为新项目模板 |
| [Apache Maven Getting Started Guide](https://maven.apache.org/guides/getting-started/) | Maven 通过统一项目结构和最佳实践管理构建、文档、依赖、发布等 | Maven 章节要围绕 `pom.xml`、生命周期、依赖范围和可重复构建来学 |
| [JUnit User Guide](https://docs.junit.org/6.1.1/overview.html) | JUnit Platform 是 JVM 测试框架基础，支持多种测试引擎和命令行运行 | 学习验收不只看页面能启动，还要保留测试输出和失败用例 |

## 旧材料的使用边界

### 仍然适合按原文学习

- Java 基础语法、类、接口、集合、异常、IO、日期时间。
- Maven 的项目结构、依赖、生命周期和常用命令。
- HTTP、Servlet、请求响应、Cookie、Session 的底层概念。
- Spring IoC、DI、AOP、Bean、Controller、Service、Mapper 的职责边界。
- SQL 参数、事务、Mapper XML、DO/DTO/VO 的工程概念。

### 适合作为历史项目阅读材料

- JSP、传统 Servlet 页面案例。
- XML 大量配置的 Spring 项目。
- Hystrix、Zuul、Swagger2、FastDFS 等旧项目常见组件。
- 基于截图的微服务商城实战。

阅读这些内容时，重点放在“它解决什么问题、边界在哪里、现在会用什么替代思路”，不要直接复制为新项目方案。

### 新文章应补齐的标准

每篇新整理的文章至少包含：

- 适用场景：什么时候需要这个知识点。
- 最小代码：能独立运行或被项目调用。
- 调用链：谁调用谁，数据怎么流动。
- 验收命令：例如 `java -version`、`mvn test`、`curl` 请求或接口测试。
- 常见错误：至少一个真实报错和定位方法。
- 版本说明：只在影响行为时写版本，不随手写“最新版”。

## 技术词第一次出现时这样写

先写能做什么，再写术语。

| 弱写法 | 更适合本系列的写法 |
|--------|--------------------|
| 学习 IoC 和 DI | 让对象依赖由 Spring 创建和组装（IoC / DI），减少类之间自己 new 的耦合 |
| 学习 AOP | 把日志、事务、权限这类横切逻辑从业务代码里抽出来（AOP） |
| 学习 Mapper | 用接口和 SQL 文件把 Java 方法映射到数据库操作（Mapper） |
| 学习 DTO 和 VO | 把数据库对象和接口返回对象分开，避免把表结构直接暴露给前端（DTO / VO） |
