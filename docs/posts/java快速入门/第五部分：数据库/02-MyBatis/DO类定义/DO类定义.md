### 一、public 类
- **定义**：在 Java 中，`public` 类意味着这个类是公开的，可以被任意其他类访问。
- **作用**：`public` 类通常作为数据模型、服务接口等在整个应用程序中共享和访问的类。

### 二、使用以下注解
这些注解通常用在数据模型（特别是实体或 DTO）上，以简化代码编写。

- **@Data**
- **库**：Lombok
- **作用**：自动为类生成所有字段的 getter 方法，为非 final 字段生成 setter 方法，还会生成 `toString`、`equals` 和 `hashCode` 方法。
- **优点**：减少样板代码，使得类更加简洁。

- **@EqualsAndHashCode(callSuper = true)**
- **库**：Lombok
- **作用**：生成 `equals` 和 `hashCode` 方法的实现，并且通过 `callSuper = true` 属性指示 Lombok 包含超类（父类）的字段在比较和计算哈希值时。
- **场景**：当你的类继承了其他类，并且你想在比较和哈希计算时包含父类的字段。

- **@ToString(callSuper = true)**
- **库**：Lombok
- **作用**：生成 `toString` 方法的实现，并且通过 `callSuper = true` 属性指示 Lombok 包含超类（父类）的字段在字符串表示中。
- **场景**：当你需要打印或日志记录类的信息，并且也想包括父类的字段信息。

### 三、字段中使用以下注解
这些注解通常用在 ORM（对象关系映射）框架中，如 MyBatis-Plus。

- **@TableId**
- **库**：MyBatis-Plus
- **作用**：标记一个字段为数据库表的主键字段。
- **场景**：用在实体类的字段上，用来指示这个字段是对应数据库表的主键列，MyBatis-Plus 会用这个字段作为查询、更新、删除操作的依据。

- **@TableField**
- **库**：MyBatis-Plus
- **作用**：将实体类的字段映射到数据库表的列。
- **场景**：当实体类的字段名和数据库表的列名不一致时，可以使用这个注解来指定映射关系。也可以用来指示某些字段不是数据库表的列。

使用这些注解可以大大简化实体类的编写，并且自动处理很多原本需要手动编写的方法，使得代码更加简洁并减少出错的概率。同时，它们也为类与数据库表之间的映射提供了简便的配置方法。
