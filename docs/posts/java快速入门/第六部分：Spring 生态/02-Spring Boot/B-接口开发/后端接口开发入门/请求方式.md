`@RequestMapping`
这种支持任意请求方式，类似于自适应。

![Mapping public interface RequestMapping](Exported%20image%2020260702230548-0.png)

`@GetMapping`
客户端只能用 `GET` 方式请求，适用于查询数据

![RequestMappingmethod RequestMethod . GET public in...](Exported%20image%2020260702230555-1.png)

`@PostMapping`
客户端只能用 `POST`方式请求，适用于提交数据。

![RequestMappingmethod RequestMethod . POST public i...](Exported%20image%2020260702230558-2.png)

`@DeleteMapping`
客户端只能用 `DELETE`方式请求，使用于删除数据。

![RequestMethod . DELETE public interface DeleteMapp...](Exported%20image%2020260702230600-3.png)

`@PutMapping`
客户端只能用 `PUT`方式请求，使用于修改数据（但在实际使用中，我个人建议还是采用`POST`方式较为妥当）。

![RequestMethod . PUT public interface PutMapping](Exported%20image%2020260702230602-4.png)

以上请求我是在接口开发中经常使用的，图片是注解源码。当然还有其他一些。关于请求方式及使用范围，可以参考 `RESTful API`
