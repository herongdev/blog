根据您提供的 logback.xml 配置文件，您可以将日志文件夹从 /home/ms/logs 更改为您用户目录下的 ms/logs 文件夹。请按照以下步骤进行操作：

将配置文件中的日志存放路径更改为 ${user.home}/ms/logs。将以下行：
xml
Copy
\<property name="log.path" value="/home/ms/logs" /\>
替换为：

xml
Copy
\<property name="log.path" value="${user.home}/ms/logs" /\>
${user.home} 是一个Java系统属性，它将被替换为您的用户目录（/Users/your_username）。

保存并关闭 logback.xml 配置文件。

创建新的日志文件夹。在终端（Terminal）中运行以下命令：

bash
Copy
mkdir -p ~/ms/logs
这将在您的用户目录下创建一个名为 ms 的文件夹，以及一个名为 logs 的子文件夹。

重新启动您的应用程序，以使更改生效。
现在，您的应用程序应该将日志文件写入新创建的日志文件夹（/Users/your_username/ms/logs）。如果您遇到任何问题，请随时告诉我，我会竭诚为您提供帮助。
