# 01-课程路线图

你最终要做出一个可运行的小型 Java 后端项目：一个任务管理 API。它包含任务创建、查询、状态更新、数据库持久化、参数校验、统一错误返回、单元测试和基础日志。

这个项目不大，但它覆盖企业后端最常见的工作动作：读项目结构、改接口、查数据库、处理异常、写测试、解释分层边界。

## 你要拿到的能力证据

| 能力 | 你要交付的证据 |
|------|----------------|
| Java 基础能跑通 | `HelloJava.java`、数组和集合练习、命令行输出截图或日志 |
| 面向对象能解释 | `Task`、`TaskService`、`TaskRepository` 的职责说明 |
| API 能用对 | `List`、`Map`、`String`、`LocalDateTime`、文件读写练习 |
| Maven 能管理项目 | `pom.xml`、`mvn test` 输出、依赖用途说明 |
| Spring Boot 能开发接口 | `TaskController`、`TaskService`、`TaskMapper` 或 Repository |
| 数据库能落地 | 建表 SQL、CRUD 示例、分页或条件查询记录 |
| 错误能复盘 | 异常堆栈、定位过程、最终修复方案 |
| 质量能自证 | 测试用例、验收清单、学习日志 |

## 阶段 0：配好能跑的 Java 环境

目标：在固定练习目录里运行第一个 Java 程序，确认 JDK、终端、IDE 和文件路径都可控。

阅读：

- [Jdk、jre和jvm](./第一部分：语言基础/00-开发环境/Jdk、jre和jvm.md)
- [java有哪些版本](./第一部分：语言基础/00-开发环境/java有哪些版本.md)
- [Java语言开发环境搭建](./第一部分：语言基础/00-开发环境/Java语言开发环境搭建/Java语言开发环境搭建.md)

动作：

```bash
cd ~/java-quickstart-course/playground
mkdir -p hello-java
cd hello-java
java -version
javac -version
```

验收：

- 你能说清楚 JDK、JRE、JVM 的区别。
- 你知道当前终端使用的是哪个 Java 版本。
- 你能解释为什么项目要固定工作目录。

## 阶段 2：把过程代码改成对象协作

目标：把成绩统计程序改造成 `Student`、`ScoreRecord`、`ScoreService` 这样的对象协作。

阅读：

- [面向对象思想概述](./第二部分：面向对象/01-类与对象/面向对象思想概述.md)
- [类和对象](./第二部分：面向对象/01-类与对象/类和对象/类和对象.md)
- [4.构造函数](./第二部分：面向对象/01-类与对象/4.构造函数.md)
- [封装](./第二部分：面向对象/01-类与对象/封装/封装.md)
- [接口](./第二部分：面向对象/03-面向对象特性/接口.md)

练习：

- `Student` 只保存学生信息。
- `ScoreRecord` 保存学生和分数。
- `ScoreService` 负责统计，不直接打印。
- `Main` 负责组装数据并输出。

验收：

- 你能说清楚一个类负责什么、不负责什么。
- 你能解释为什么服务类不应该直接依赖控制台输入。
- 你能把“新增等级统计”加进去，而不把所有代码塞回 `main`。

## 阶段 5：做出 Spring Boot 任务 API

目标：把命令行任务程序升级成 HTTP API。

阅读：

- [SpringBoot快速入门](./第六部分：Spring 生态/02-Spring Boot/A-入门/SpringBoot快速入门/SpringBoot快速入门.md)
- [返回写死数据](./第六部分：Spring 生态/02-Spring Boot/B-接口开发/后端接口开发入门/返回写死数据.md)
- [请求方式](./第六部分：Spring 生态/02-Spring Boot/B-接口开发/后端接口开发入门/请求方式.md)
- [接收参数](./第六部分：Spring 生态/02-Spring Boot/B-接口开发/后端接口开发入门/接收参数.md)
- [返回格式](./第六部分：Spring 生态/02-Spring Boot/B-接口开发/后端接口开发入门/返回格式.md)
- [全局异常处理](./第六部分：Spring 生态/02-Spring Boot/B-接口开发/全局异常处理.md)

最小接口：

| 方法 | 路径 | 行为 |
|------|------|------|
| `POST` | `/tasks` | 创建任务 |
| `GET` | `/tasks` | 查询任务列表 |
| `GET` | `/tasks/{id}` | 查询单个任务 |
| `PATCH` | `/tasks/{id}/status` | 修改任务状态 |

验收：

- Controller 只处理 HTTP 输入输出。
- Service 负责业务规则。
- Repository 或 Mapper 负责数据访问。
- 异常返回格式统一。
