在 macOS 中，/home 目录是自动装载点（automount point），并且被系统保留为 NFS（网络文件系统）使用。当你尝试在这个目录下创建文件或目录时，会收到 "Operation not supported" 的错误。

对于大多数 macOS 用户来说，用户的家目录是在 /Users/ 下，而不是 /home/。虽然 /home/ 在文件系统树中存在，但是它的用途与 Linux 中的 /home/ 并不相同。这就是为什么在尝试在 /home/ 下创建目录时，你会收到错误。

因此，你应该在 /Users/ 下创建你的目录。你可以运行以下命令：

bash
Copy code
mkdir -p /Users/$(whoami)/hzz/dcdb/2023/07
这将在你的用户目录下创建所需的目录。$(whoami) 是一个会被你的用户名替换的 shell 变量。然后在你的应用程序中，你需要使用新的路径，即：/Users/你的用户名/hzz/dcdb/2023/07。

如果你确实需要在 /home/ 下创建目录（例如，如果你有特殊的需求或设置），你可能需要更改系统设置或使用其他高级技术，这通常需要更深入的系统知识，并可能需要采取措施避免破坏系统。在大多数情况下，我建议你使用 /Users/你的用户名/ 路径，而不是尝试在 /home/ 下创建目录。
