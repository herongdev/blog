对于这个`Spring Boot`项目，你可以在应用代码级别配置`CORS`。这意味着你不必在`Nginx`中配置`CORS`。

`Spring`框架提供了一种非常方便的方式来处理`CORS`。以下是如何在`Spring`项目中进行配置：

全局`CORS`配置
你可以在一个`Spring MVC`的`Java`配置中添加一个全局`CORS`配置。在你的`Spring Boot`应用中，可以创建一个新的`@Configuration`类，然后添加`@Bean`方法来实现`WebMvcConfigurer`。示例如下：

java
Copy code
@Configuration
public class MyConfiguration {

    @Bean
    public WebMvcConfigurer corsConfigurer() {
        return new WebMvcConfigurer() {
            @Override
            public void addCorsMappings(CorsRegistry registry) {
                registry.addMapping("/**")
                        .allowedOrigins("*")
                        .allowedMethods("GET", "POST", "PUT", "DELETE")
                        .allowedHeaders("*")
                        .allowCredentials(true).maxAge(3600);
            }
        };
    }
}
 在上面的配置中，`addMapping("/**")`指定了对于所有的请求都允许跨域请求。`allowedOrigins("*")`指定了所有的源都允许访问。`allowedMethods("GET", "POST", "PUT", "DELETE")`指定了允许的方法。`allowedHeaders("*")`指定了允许的请求头。

控制器级`CORS`配置
如果你只想为特定的`Spring MVC`控制器（或控制器方法）启用`CORS`，你可以使用`@CrossOrigin`注解。下面是一个简单的例子：

java
Copy code
@CrossOrigin(origins = "*", allowedHeaders = "*", methods = {RequestMethod.GET, RequestMethod.POST, RequestMethod.PUT, RequestMethod.DELETE})
@RestController
public class MyController {
    // ...
}
 在上面的例子中，`@CrossOrigin`注解应用到了`MyController`类。这表示所有这个控制器的方法都允许`CORS`。

注意：虽然在代码级别配置`CORS`更灵活，更易于管理，但是它也有一些缺点。例如，如果你的`Spring`应用程序在多个服务器上运行，你需要在每个服务器上都进行相同的配置。同时，如果你的应用程序使用了一些不能修改源码的第三方库，那么你可能无法在代码级别配置`CORS`。在这些情况下，你可能需要在`Nginx`级别配置`CORS`。
