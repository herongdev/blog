反思
由于`BaseMapper`提供的`2`个更新方法都是传入一个实体对象去执行更新，这在需要更新的列比较多时还好，若想要更新的只有那么一列，或者两列，则创建一个实体对象就显得有点麻烦。针对这种情况，`UpdateWrapper`提供有`set`方法，可以手动拼接`SQL`中的`SET`语句，此时可以不必传入实体对象，示例如下
